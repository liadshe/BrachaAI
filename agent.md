# Bracha AI — Frontend Agent Instructions

# Project Overview

The application is called **Bracha AI**.

The goal of the application is to act as an AI-powered personal assistant for small businesses.

The system records phone calls, converts speech to text, processes the information using AI, creates tasks automatically, manages contacts, and provides additional business insights.

The project is divided into 3 main parts:

1. Android Application
2. Backend System
3. Frontend UI Application

The Android application and Backend system are already implemented.

The current goal is to implement the Frontend UI.

---

# 1. Android Application

## Description

The Android application listens to incoming phone calls on the device and automatically processes audio.

### Key Components

1. **`CallMonitorService`** - Foreground service that watches for new audio files in the device storage directory using `FileObserver`. Runs persistently with notification and starts `AudioProcessor` for each new file.

2. **`AudioProcessor`** - Orchestrates the complete pipeline:
   - Parses audio filename in format `ContactName_YYMMDD_HHMMSS.ext` using `FilenameParser`
   - Converts audio to standardized MP3 (128k bitrate, 44100Hz, stereo) using FFmpeg
   - Sends MP3 to OpenAI Whisper API for Hebrew speech-to-text transcription
   - POSTs transcription result to Node.js backend at `http://10.0.2.2:3000/api/calls`

3. **`WhisperApiClient`** - Handles communication with OpenAI Whisper API (model `whisper-1`, language `he` for Hebrew). Sends audio as multipart form data to avoid encoding issues with Hebrew filenames.

4. **`MainActivity`** - Jetpack Compose UI that:
   - Requests necessary permissions (READ_MEDIA_AUDIO, POST_NOTIFICATIONS for Android 13+)
   - Provides ALL_FILES_ACCESS permission for Android 11+
   - Contains button to start the `CallMonitorService`

5. **`FilenameParser`** - Parses audio filenames safely from right to left, handling contact names with underscores

### Technical Details

- **Minimum SDK**: 26 (Android 8.0)
- **Target SDK**: 36
- **Backend URL**: `http://10.0.2.2:3000/api/calls` (Android emulator's localhost alias)
- **FFmpeg Library**: `ffmpeg-kit-audio` (audio-only variant, runs synchronously)
- **Network**: Clear HTTP traffic enabled via manifest (`usesCleartextTraffic=true`)
- **Concurrency**: Uses Kotlin coroutines on `Dispatchers.IO` for async operations
- **API Key**: OpenAI API key injected at build time from `local.properties` to `BuildConfig.OPENAI_API_KEY`

# 2. Backend System

## Description

Node.js/Express backend using TypeScript that processes transcriptions, manages data, and uses OpenAI for AI analysis.

### Architecture

**Framework**: Express.js 5.x with TypeScript
**Database**: MongoDB (Mongoose ODM)
**AI**: OpenAI API (GPT-4o for analysis)

### Key Components

1. **`index.ts`** - Express server entry point:
   - Loads environment variables from `.env`
   - Connects to MongoDB
   - Sets up CORS middleware
   - Mounts call routes at `/api` prefix
   - Listens on port 3000 (or `process.env.PORT`)

2. **`controllers/callController.ts`** - `handleIncomingAndroidCall` endpoint:
   - Receives POST requests from Android app with: `contactName`, `date`, `transcript`
   - Parses date from format `YYMMDD_HHMMSS`
   - Gets or creates contact record
   - Saves raw call data to database
   - Calls AI service for analysis
   - Creates tasks from AI-generated task list
   - Returns analysis results to client

3. **`services/aiService.ts`** - AI analysis with GPT-4o:
   - Analyzes transcript to generate:
     - **Summary**: Concise description of call content and commitments
     - **Tasks**: Array of action items with title, description, and priority
     - **Priority Rules**:
       - HIGH: Urgent/time-sensitive, near deadline, blocks important work
       - MEDIUM: Important but not urgent, clear commitment
       - LOW: Nice-to-do, vague, or informational
   - Returns structured JSON with `summary` and `tasks[]`

4. **`services/callService.ts`** - Call data management:
   - `saveRawCall()`: Saves transcription with contact and user references
   - `updateCallWithAnalysis()`: Updates call record with AI-generated summary

5. **`services/taskService.ts`** - Task creation:
   - `createTasksFromAi()`: Saves tasks generated by AI to database

6. **`services/userService.ts`** - User/contact management:
   - `getOrCreateContact()`: Finds or creates contact record

### Database Models

- **User**: `name`, `email`, `phone`, `avatar`, `permissions`
- **Contact**: `userId`, `name`, `phone`, `isVip`
- **Call**: `userId`, `contactId`, `fullTranscript`, `callSummary`, `callDateTime`, `callLength`
- **Task**: `userId`, `contactId`, `title`, `description`, `priority` (LOW|MEDIUM|HIGH), `status` (todo|in-progress|done)

# 3. Frontend UI Application

## Goal

Implement a modern web interface for the Bracha AI system using React and PWA capabilities. Users should be able to:

- Access the system from any browser
- View and manage call history and transcriptions
- View AI-generated summaries and insights
- Manage tasks created from calls
- View and manage contacts
- Install the app on phone home screen as a native app (PWA)
- Use offline capabilities with service workers

## Tech Stack

- **Framework**: React 18+
- **Language**: TypeScript
- **Styling**: CSS-in-JS or Tailwind CSS
- **State Management**: Context API or Redux
- **PWA**: Service Workers, Web App Manifest
- **HTTP Client**: Axios or Fetch API
- **UI Components**: Material-UI or custom components
